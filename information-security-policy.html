This Information Security Policy describes the technical and organizational measures Pocket Check LLC ("we," "us," or "our") uses to protect the data you share with us through the Pocket Check mobile application (the "App"). This policy supplements our Privacy Policy.

1. Overview

Security is a core principle of how Pocket Check is built. Because we handle sensitive financial data, we apply a defense-in-depth approach — multiple layers of security controls designed to protect your information even if any single control fails.

Our security design is guided by three principles:

  • Minimal data collection: We only collect what is necessary to deliver the service
  • Local-first storage: Sensitive financial data lives on your device, not our servers
  • Read-only access: We cannot move, modify, or transact on your bank accounts under any circumstances

2. Data Transmission Security

All data transmitted between the App and our backend servers, and between our servers and third-party services, is protected by:

  • TLS (Transport Layer Security) encryption on all network connections
  • Certificate validation to prevent man-in-the-middle attacks
  • HTTPS enforced on all API endpoints — unencrypted HTTP connections are rejected

Your bank credentials are never transmitted to Pocket Check. Authentication with your bank is handled entirely by Plaid through their secure, encrypted connection.

3. Data Storage Security

3.1 On-Device Storage

Financial data stored on your device is protected by:

  • iOS encrypted secure storage (Keychain) for sensitive values
  • Device-level encryption enforced by iOS when your device is locked
  • No backup of financial data to iCloud or other cloud storage services

3.2 Server-Side Storage

The only data we store on our backend is your Plaid access token and, if applicable, your push notification token. These are protected by:

  • Encrypted storage within our Railway-hosted database
  • Environment-level access controls restricting who and what can read the database
  • No plaintext storage of any credentials or sensitive tokens

4. Access Controls

Access to systems and data that power Pocket Check is strictly limited:

  • Server infrastructure access is restricted to authorized personnel only
  • All administrative access requires strong authentication
  • Our backend API enforces authentication on all endpoints that access user data
  • API keys and secrets are stored as environment variables, never hardcoded in source code
  • Access permissions follow the principle of least privilege — each system component can only access what it needs

5. Third-Party Security

We carefully select third-party services and hold them to security standards consistent with our own:

Plaid Technologies, Inc.

Plaid is a SOC 2 Type II certified financial data provider. They use bank-grade encryption and never share your login credentials with Pocket Check. Plaid's security practices are described at plaid.com/safety.

Anthropic (Claude AI)

Chat messages and financial context sent to Anthropic's API are transmitted over encrypted connections. Anthropic does not retain request data beyond the duration of each API call. Learn more at anthropic.com/privacy.

Railway

Our backend is hosted on Railway, which provides infrastructure-level security including encrypted storage and network isolation. Learn more at railway.app/legal/privacy.

6. Incident Response

In the event of a security incident that affects your data:

  • We will investigate and contain the incident as quickly as possible
  • We will notify affected users within a reasonable timeframe, consistent with applicable law
  • We will take steps to prevent recurrence and improve our controls

To report a suspected security vulnerability or incident, contact us immediately at pocketcheckapp@gmail.com. We take all security reports seriously and will respond promptly.

7. Your Role in Security

Security is a shared responsibility. You can help protect your data by:

  • Keeping your device's operating system and the Pocket Check app up to date
  • Using a strong device passcode or biometric authentication
  • Not sharing your device with others while the App is open
  • Disconnecting your bank account from within the App if you change devices or sell your phone
  • Reporting any unusual behavior in the App to us promptly

8. Limitations

Despite our best efforts, no security system is perfect. We cannot guarantee absolute security of data transmitted over the internet or stored on electronic systems. By using the App, you acknowledge this inherent risk.

We are not responsible for security breaches caused by factors outside our reasonable control, including failures in third-party infrastructure, vulnerabilities in the iOS operating system, or unauthorized access to your physical device.

9. Changes to This Policy

We may update this policy from time to time to reflect changes in our practices or applicable law. When we do, we will update the "Last Updated" date at the top of this page. For significant changes, we will notify you through the App.
